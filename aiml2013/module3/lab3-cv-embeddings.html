The Task

In the readings and in class, you learned that HOG encodes local edge orientations: a fixed recipe designed by researchers. It works when objects share an edge structure and fails when they do not. A neural network trained on millions of labeled images learns its own features, and those features encode higher-level patterns (textures, parts, shapes) that no hand-designed formula can capture.

This week you extract embeddings from a pre-trained MobileNetV2, run the same similarity retrieval task from Module 2 on the same images, and produce a direct comparison. The cells where HOG failed are where this lab gets interesting. You will find out exactly what the network learned that HOG could not represent, and you will find the cases where even learned features still struggle.

# Cell 3: Preprocess and extract MobileNetV2 embeddings

What it does: Resizes every image to 224×224 (MobileNetV2’s expected input size), applies the model’s preprocessing function, passes the batch through the network, and stores the output embeddings. Prints the shape and a sample vector.

Takes in: feature_model, images, preprocess_input (from Cells 1–2).

Must produce:

  • For each image in images, resize it to (224, 224) using tf.image.resize(image, (224, 224)) or skimage.transform.resize(image, (224, 224)). If using TensorFlow, call .numpy() on the result to convert to a numpy array. Collect all resized images into a single numpy array of shape (n_images, 224, 224, 3) with dtype float32.
  • Apply preprocess_input(batch_array) to the entire batch. This function scales pixel values to the range that MobileNetV2’s pre-trained weights expect (typically centered around zero, with a standard deviation of roughly 1). Do not apply preprocessing to individual images one at a time — preprocess the whole batch array.
  • Call feature_model.predict(preprocessed_batch, verbose=0) on the entire preprocessed batch. This is much faster than calling predict() on individual images. The output is a numpy array of shape (n_images, 1280).
  • embed_matrix: numpy array, shape (n_images, 1280), dtype float32. This is the direct output from feature_model.predict(). Row i is the MobileNetV2 embedding vector for image i.
  • Printed output: (1) the matrix shape, (2) the first 10 values of embed_matrix[0] rounded to 4 decimal places, and (3) the percentage of zero values in the entire matrix, computed as (embed_matrix == 0).mean() * 100. (MobileNetV2 uses ReLU activations, which set negative values to zero, so sparsity is common.)

Notes: Batched prediction is critical for speed. For 150 CIFAR images, batched prediction takes about 5 seconds; doing it one image at a time would take much longer. Always build the full batch before calling predict().
