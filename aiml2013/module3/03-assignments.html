Deliverable 1: Module 3 Lab Lab

Overview

In the readings and in class, you learned that a convolutional neural network trained on ImageNet learns to organize images into a hierarchy of features: edges in early layers, textures and shapes in middle layers, object parts in deep layers. This lab strips the classification head off MobileNetV2 and uses what remains as a feature extractor, producing a 1,280-dimensional embedding for each image. You'll build a similarity retrieval system, compare it head-to-head against your Module 2 HOG baseline, and use t-SNE to visualize how the embedding space clusters images.

The HOG comparison is the point. Both systems retrieve images by computing distances between feature vectors. What changes is what the feature vector encodes: a hand-designed gradient histogram versus a representation learned from 1.2 million labeled images. Finding where the gap is real, and finding where it isn't, is what the lab is about.

About This Lab

This lab has no code scaffold. You write every cell yourself, using an LLM to help you generate the implementation. The lab sheet gives you specification boxes: each one tells you exactly what a cell must do, what it takes in, and what it must produce. Your job is to give those specifications to Gemini (or another LLM) and turn the output into working code.

For each code cell you generate, include a markdown cell documenting the prompt you used and one sentence describing any fix you had to make. This process is the core skill of the module.

What to Submit

  • A completed Colab notebook (.ipynb) saved to your GitHub repo at week4/lab3_embeddings.ipynb.
  • All cells executed with visible outputs (images must render in GitHub's notebook viewer).
  • Each code cell preceded by a markdown cell with your prompt and fix note.
  • No API keys or secrets anywhere in the notebook.
  • Submit the GitHub link in Canvas. The notebook in GitHub IS your submission — if it's not in the repo, it doesn't exist for grading.
Start this in Thursday lab. We'll work through the MobileNetV2 feature extraction pipeline together: loading the model without the classification head, running images through it, and building the similarity retrieval system. Come with the readings done so you already know what global average pooling produces before you write the first cell.

Deliverable 2: Module 3 Demo Demo

Overview

A three- to five-minute live demo during Tuesday's class (April 14, 7:00 PM). Your notebook is your demo — no separate slides required. Walk the class through your retrieval results, make a live modification, and explain what changed.

What to Cover

In three to five minutes, walk the class through a live demonstration:

  1. The win: Show a query image where MobileNetV2 retrieved semantically similar images that HOG missed. Navigate to that cell in your notebook and explain why the learned embedding got it right when the gradient histogram didn't.
  2. The t-SNE plot: Show your visualization and point to one cluster that surprised you. Explain what the model considers "similar" that you wouldn't have predicted.
  3. Live modification: Swap a query image or adjust a parameter, run the cell live, and explain what changed and why. This is the core of the demo — don't skip it.
  4. Your takeaway: Based on your results, what does the shift from HOG to learned embeddings buy you, and what does it cost?

Format

Share your screen and run your notebook live. If you attend via Zoom, be prepared to share your screen and have your notebook open and ready to run.

Tip: The dual retrieval comparison (HOG vs. MobileNetV2, same query image, side by side) is your strongest visual. Pick a query where the gap is obvious — that's what you'll modify live.
