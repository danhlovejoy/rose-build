Deliverable 1: Module 5 Lab Lab

Overview

In the readings and in class, you learned that a classifier's overall accuracy can hide large disparities across demographic subgroups. The Gender Shades study showed this at industrial scale: commercial face classifiers that appeared to work well overall were failing on specific populations. This lab builds your own version of that audit. You will train a gender classifier on CelebA, compute overall metrics, disaggregate by demographic attributes, run an intersectional analysis, and produce a bias scorecard.

The failure gallery in Cell 8 is where the audit becomes real. Numbers tell you the model is wrong 20% of the time on a subgroup; the images show you the specific faces it gets wrong and the visual patterns behind the errors.

What to Submit

  • A completed Colab notebook (.ipynb) saved to your GitHub repo at week6/lab5_bias_audit.ipynb.
  • All cells executed with visible outputs (don't clear outputs).
  • Each code cell preceded by a markdown cell with your LLM prompt and fix note.
  • No API keys or secrets anywhere in the notebook.
  • Submit the GitHub link in Canvas. If it's not in the repo, it doesn't exist for grading.
Start this in Thursday lab. We'll work through the audit pipeline together: loading CelebA, training the classifier head, and computing the first per-group metrics. Come with the readings done — understanding what Gender Shades found will help you interpret your own results.

Deliverable 2: Module 5 Demo Demo

Overview

A three- to five-minute live demo during Tuesday's class (April 28, 7:00 PM). Your notebook is your demo — no separate slides required. Walk the class through your bias audit, show where the model fails, and make a live modification.

What to Cover

  1. Overall metrics (30 seconds): Show the confusion matrix and classification report. Orient the audience on how the model performs before disaggregation.
  2. The disaggregated results: Show the per-group bar chart from Cell 4. Which subgroup performs worst? How large is the gap compared to overall accuracy?
  3. Intersectional analysis: Show the 2×2 heatmap from Cell 5. Did the intersection reveal a failure that single-attribute analysis missed?
  4. The failure gallery: Show Cell 8. Pick two misclassified images and explain what visual features might have confused the classifier.
  5. Live modification: Change something — the classification threshold, the slicing attributes, or the intersectional combination. Re-run and explain what changed. This is worth 25 points on the demo rubric.

Format

Share your screen and run your notebook live. If you attend via Zoom, have your notebook open and ready to run before your turn.
