The Task

In AIML 2003, you have been working with text. In AIML 2013, you have been working with images. This week you will build the same pipeline twice: once for each modality, and compare what you find.

The pipeline: take raw input (a sentence, or a photograph), reduce it to a fixed-length feature vector, then use that vector to retrieve the most similar items from a corpus. You will run each pipeline, find where it succeeds, and, more importantly, find where it fails. The failures are the insight.

In the readings and in class, you learned that both language and vision pipelines face the same fundamental problem: how do you turn something a human understands into numbers a machine can compute on? TF-IDF and HOG are two answers, both hand-crafted, both transparent, and both limited in exactly the same direction. By the end of this lab, you will know what that shared limitation is.

# Cell 3: Build TF-IDF feature vectors

What it does: Fits a TF-IDF vectorizer on the full documents list and transforms it into a sparse matrix. Prints the matrix shape and sparsity percentage. Then prints the top 10 TF-IDF terms for the very first document.

Notes: A sparse matrix stores only nonzero values, saving memory. The .nnz attribute gives the count of nonzero values. Most reviews will have zero weight for most of the 5,000 terms.
