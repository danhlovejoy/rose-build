Getting Started with Google Colab

AIML 2013: Setup Guide. Your coding environment for the course

Google Colab (short for "Colaboratory") is a free, browser-based notebook environment where you can write and run Python code. It runs on Google's servers, so you don't need to install anything on your own computer. All of our labs will be built in Colab.

What You Need

A Google account. If you already use Gmail, Google Drive, or any Google service, you're set. If you don't have one, create a free account at accounts.google.com/signup.

Opening Google Colab

1

Go to Google Colab

Navigate to colab.research.google.com. Sign in with your Google account if prompted.

2

Create a new notebook

Click New Notebook (or go to File → New notebook). A new notebook will open with an empty code cell ready to go.

3

Run your first cell

Type the following into the code cell and press Shift + Enter (or click the play button) to run it:

print("Hello, Computer Vision!")

If you see the output below the cell, you're up and running.

4

Rename your notebook

Click on "Untitled0.ipynb" at the top left and rename it to something descriptive, like week1_setup.ipynb. This makes it easier to find later in Google Drive.

Key Concepts

Cells

Notebooks are made up of cells. Code cells are where you write Python. Text cells are where you write notes in Markdown. You can add new cells with the + Code or + Text buttons at the top.

Runtime

Your code executes on a virtual machine hosted by Google (called your "runtime"). If you're idle for ~90 minutes or your session exceeds ~12 hours, the runtime will disconnect and you'll lose variables in memory. Your notebook file stays safe on Google Drive though.

Installing packages

Most common Python libraries (numpy, pandas, etc.) are pre-installed. To install something else, run this in a code cell:

!pip install google-genai

The ! prefix tells Colab to run the command in the system shell, not as Python.

Keeping Your API Key Safe

Critical: In this course, you'll use API keys to access LLMs. Never paste your API key directly into a code cell. If you accidentally commit a notebook with an exposed key to GitHub, bots will find and steal it within minutes.

Use Colab's Secrets Manager (recommended)

Colab has a built-in way to store secrets securely:

  1. Click the key icon in the left sidebar (or go to Tools → Secrets).
  2. Click Add new secret.
  3. Name it GEMINI_API_KEY and paste your key as the value.
  4. Toggle "Notebook access" to ON for the current notebook.

Then access it in your code like this:

from google.colab import userdata
api_key = userdata.get('GEMINI_API_KEY')

This way, your key never appears in the notebook file itself.

Tip: We'll walk through API key setup together during the Week 1 Thursday lab. Don't worry if this doesn't make sense yet; just know that there's a safe way to do it.

Useful Keyboard Shortcuts

These will save you a lot of time throughout the course:

  • Shift + Enter: Run the current cell and move to the next one
  • Ctrl + Enter: Run the current cell and stay on it
  • Ctrl + M, B: Insert a new code cell below
  • Ctrl + M, M: Convert a cell to Markdown (text)
  • Ctrl + M, D: Delete the current cell
  • Ctrl + /: Comment/uncomment selected lines

Saving Your Work

Your notebooks auto-save to Google Drive in a folder called "Colab Notebooks." You can also save manually with Ctrl + S. To download a local copy, go to File → Download → Download .ipynb.

Later in this module, you'll learn how to connect Colab directly to GitHub so your work is version-controlled from the start.

Optional: Using Colab from VS Code

If you prefer working in VS Code instead of the browser, Google's official Colab extension lets you run notebooks locally in VS Code while using Colab's cloud runtimes (including GPUs). This is optional. The browser works fine for everything in this course, but VS Code gives you better code editing, Git integration, and access to your full extension library.

1

Install the extension

Open VS Code, go to Extensions (Ctrl/Cmd+Shift+X), search for "Google Colab", and install the official extension by Google. It will also install the Jupyter extension if you don't have it.

2

Connect to a Colab runtime

Open any .ipynb file in VS Code, then click Select Kernel in the top right. Choose Colab and select your runtime type. Sign in with your Google account when prompted.

3

Run cells as usual

Your code runs on Colab's cloud servers, but you edit in VS Code with all its features: autocomplete, terminal, debugging, and Git built in.

Limitation: Colab's Secrets Manager (where we store API keys) is not available through the VS Code extension yet. If you use VS Code, you'll need to store your GEMINI_API_KEY in a local .env file (and add .env to your .gitignore so it never gets committed). We'll cover this in Thursday lab for anyone who wants to use VS Code.

Common Questions

Do I need a powerful computer?

No. Colab runs in the cloud. Any device with a modern web browser will work; even a Chromebook.

Is Colab really free?

Yes. The free tier is more than enough for everything in this course. You do not need Colab Pro.

What if my runtime disconnects?

Your notebook file is safe (it's on Drive). You'll just need to re-run your cells from the top. Get in the habit of writing code that can be re-run cleanly.

Can I use a local Jupyter notebook instead?

For labs, please use Colab so we're all on the same environment and I can help debug issues. You're welcome to also work locally if you prefer, but submit the Colab version.
