The Task

In the readings and in class, you learned that vision-language models (VLMs) combine image encoders with language decoders to generate text descriptions of visual content. Unlike the classifiers you've used so far (which output a label or a score), VLMs produce open-ended text. This makes them powerful — they can describe anything in an image — and hard to evaluate, because there is no single correct caption for any image.

You will use Gemini's vision capabilities to generate captions for images, then evaluate those captions against human-written reference descriptions. The evaluation is the core of this lab: you will measure whether the model describes what's actually in the image, whether it adds objects that aren't there (hallucinated objects), and how caption quality varies across different image types.

By the end, you will have a captioning evaluation pipeline and a clear understanding of where VLMs succeed and where they fabricate.

Dataset

You will use a curated set of 30 images from multiple categories. For each image, you write a brief reference description (1–2 sentences) before running the model, then compare Gemini's caption to your reference.

Upload images to Colab via the Files panel or load from a URL. Resize each to have a longest side of 800 pixels before sending to Gemini (saves API quota).

Part 2: Evaluate Caption Quality

# Cell 4: Object Accuracy — What's Real and What's Hallucinated

What it does: For each image, extracts the objects mentioned in the detailed caption and compares them to the objects in the reference description. Classifies each mentioned object as correct (in the image), hallucinated (not in the image), or missed (in the reference but not the caption).

Takes in: client, MODEL_ID, image_data.

Must produce:

  • For each image, use Gemini as a judge: send it the image, the model's detailed caption, and your reference description. Ask it to output a JSON object with three lists: correct_objects, hallucinated_objects, missed_objects.
  • eval_results: a list of 30 dictionaries, each with keys: 'category', 'n_correct', 'n_hallucinated', 'n_missed', 'hallucinated_objects' (list of strings), 'precision' (correct / (correct + hallucinated)), 'recall' (correct / (correct + missed)).
  • Print: overall hallucination rate (total hallucinated / total objects mentioned), overall precision, overall recall.

Notes:

  • The LLM-as-judge technique from Module 5 works here too. Prompt Gemini carefully: "You are an image analysis judge. Given the image, a model's caption, and a human reference, list which mentioned objects are actually in the image (correct), which are NOT in the image (hallucinated), and which are in the reference but missing from the caption (missed). Output ONLY valid JSON."
  • Parse the JSON response. If parsing fails (the model sometimes wraps JSON in markdown code blocks), strip the backticks and retry.
  • This is the same model judging its own captions. The circular evaluation problem from Module 5 applies. Note this limitation in your reflection.
Markdown cell: What was your overall hallucination rate? Was it higher or lower than you expected? Did certain categories of images produce more hallucinated objects than others?

# Cell 7: Prompt Sensitivity Test

What it does: Tests whether the captioning prompt affects hallucination rate. Runs a subset of images with three different prompts and compares the results.

Takes in: client, MODEL_ID, image_data.

Must produce:

  • Select 10 images (2 from each of 5 categories).
  • Run each image with three prompts:
    1. Minimal: "What is in this image?"
    2. Cautious: "Describe only the objects you can clearly see in this image. If you are uncertain about an object, do not mention it."
    3. Exhaustive: "List every single object, person, color, texture, and detail in this image. Be as thorough as possible."
  • For each prompt, compute the average hallucination rate (using the same judge from Cell 4).
  • A bar chart comparing hallucination rate across the three prompts.
  • Print: which prompt produced the fewest hallucinations and which produced the most.

Notes: The cautious prompt should reduce hallucinations by telling the model to be conservative. The exhaustive prompt should increase them by encouraging the model to generate more content. If the cautious prompt doesn't reduce hallucinations, the model may not respond well to hedging instructions — that's an important finding.

Markdown cell: Did the cautious prompt reduce hallucinations? By how much? What does this tell you about how much control you have over a VLM's behavior through prompt engineering alone?

# Bonus Cell: VQA Test

What it does: Extends the captioning pipeline into visual question answering (VQA). Instead of describing the whole image, you ask specific questions and evaluate whether the model answers correctly.

Takes in: client, MODEL_ID, image_data.

Must produce:

  • For 10 images, write 2 questions each (20 total). One easy question (answerable from the main subject) and one hard question (requires noticing a small detail).
  • Send each image + question to Gemini. Use a judge to score each answer as correct, partially correct, or wrong.
  • Print: VQA accuracy for easy vs. hard questions. Show examples of correct and incorrect VQA answers.

Notes: VQA is the next step beyond captioning: instead of open-ended description, the model answers targeted questions. This is how VLMs are used in accessibility tools (describing specific elements of an image for visually impaired users), medical imaging (answering diagnostic questions about X-rays), and document understanding (reading text from scanned documents).
