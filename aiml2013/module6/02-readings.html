Module 6: Generative Vision

Readings & Prep: Complete before Tuesday, May 5

Estimated prep time: ~1.5 hours (required materials). One optional deep dive on VLM architectures. All materials are free.

The big picture: Every CV model you've used this semester has produced structured outputs: a class label, an embedding vector, a similarity score, an accuracy metric. This week you cross into generative territory. A vision-language model takes an image as input and produces open-ended text as output. The text might be a caption, an answer to a question, or a detailed scene description. And unlike a class label, there is no single correct answer — which makes evaluation much harder.

VLMs hallucinate. They describe objects that aren't in the image, invent spatial relationships, and add details that "make sense" for the scene type but are factually wrong. The CHAIR metric (Caption Hallucination Assessment with Image Relevance) was designed specifically to measure this. Your lab builds a simplified version: you generate captions, compare them to reference descriptions, and catalog what the model invents.

This week's readings cover three things: how vision-language models combine image encoders with language decoders, how researchers measure captioning hallucination, and how VLM accuracy matters for real applications like accessibility and medical imaging.

How Vision-Language Models See and Describe Required Video ~20 min

Computerphile • YouTube

A clear overview of how VLMs work: an image encoder (like the MobileNetV2 you used in Module 4) extracts visual features, which are projected into the same embedding space as text tokens, then fed to a language model decoder that generates text. This is the architecture behind Gemini Vision, GPT-4V, and Claude's vision capabilities. Watch for the explanation of why the model can describe objects it has never seen — the language decoder generalizes from the visual features, the same way it generalizes from text context.

Watch: Vision-Language Models Explained →

Key concept to lock in: A VLM is two models stitched together: a vision encoder that converts an image into feature vectors, and a language decoder that converts those vectors into text. The magic (and the risk) is in the stitching: the decoder generates text based on visual features, but it also draws on its language training — which is where hallucinated objects come from.
Object Hallucination in Image Captioning Required Read ~25 min

Rohrbach et al., 2018 • EMNLP

Read Sections 1–4. This paper introduced CHAIR (Caption Hallucination Assessment with Image Relevance), the standard metric for measuring how often captioning models describe objects that aren't in the image. CHAIR compares the objects mentioned in a generated caption against the objects annotated in the ground truth (COCO object labels). Your lab does the same thing, using LLM-as-judge instead of COCO annotations. The paper also categorizes hallucination patterns: objects from the same scene type (a caption about a kitchen mentions a microwave that isn't there), objects from similar categories (a caption about a dog mentions a cat), and entirely fabricated objects.

Read: Object Hallucination in Image Captioning →

Key concept to lock in: Hallucinated objects aren't random. They follow patterns: they belong to the scene type (kitchens get microwaves, beaches get umbrellas), they are visually similar to real objects (a brown pillow gets called a cat), or they fill expected slots (a dining table gets plates even when it's empty). Your lab's hallucination gallery will show these same patterns.
AI-Generated Alt Text: Promise and Pitfalls for Accessibility Required Read ~15 min

Google AI Blog • blog.google

Google uses VLMs to generate alt text for images on the web, making visual content accessible to screen reader users. This blog post describes the pipeline and the challenges: hallucinated objects are especially harmful when the user cannot see the image to verify the description. A sighted person can ignore a wrong caption; a blind person may form an entirely wrong mental model of the scene. This is the application context for your lab — caption accuracy is not an academic metric when someone depends on it.

Optional Deeper Dive

Visual Instruction Tuning (LLaVA) Optional Read ~30 min

Liu et al., 2023 • NeurIPS

LLaVA (Large Language and Vision Assistant) is one of the most influential open-source VLMs. This paper explains how visual instruction tuning works: you take a pre-trained language model, connect it to a vision encoder, and fine-tune it on image-text instruction pairs. The architecture is essentially what Gemini Vision does behind the scenes. Worth reading if you want to understand the training process, not just the inference API.

Discussion Prep

These questions will frame our Tuesday class discussion. You don't need to write answers, but come prepared to talk about them.

  1. Your lab showed that different prompts produce different hallucination rates. The "cautious" prompt reduced hallucinations but also reduced detail. If you're building an accessibility tool, should you prioritize completeness (describe everything, accept some hallucinations) or accuracy (describe only what's certain, miss some real objects)?
  2. The CHAIR paper measures hallucination against COCO object annotations. Your lab measures hallucination against your own reference descriptions. Both methods have blind spots: COCO only annotates 80 object categories, and your references reflect what you noticed. How does the choice of ground truth shape what counts as a hallucination?
  3. VLMs describe images in language, which means the caption reflects the biases of the language model. If a VLM captions an image of a person cooking as "a woman in the kitchen" when the person's gender is ambiguous, is that a hallucination or a bias? How do you distinguish between the two?
  4. Gemini Vision can generate detailed descriptions of medical images (X-rays, MRIs). Should VLMs be used in medical imaging without human radiologist review? What hallucination rate would be acceptable for a medical application?
