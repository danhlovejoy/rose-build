Deliverable 1: Module 4 Lab Lab

Overview

In the readings and in class, you learned how pre-trained CNNs serve as feature extractors: remove the classification head, feed in an image, and the remaining layers produce a dense embedding that encodes visual content. In Module 3 you computed similarity by brute-force cosine distance against a numpy array. This lab replaces that with a real retrieval system backed by ChromaDB, adds quantitative evaluation (precision@k), and runs the HOG comparison one more time to measure the gap.

The failure case analysis in Cell 8 is where the interesting findings live. Every retrieval system has blind spots. Identifying them — and explaining why the feature extractor fails on those specific images — is more valuable than reporting a high precision score.

What to Submit

  • A completed Colab notebook (.ipynb) saved to your GitHub repo at week5/lab4_similarity.ipynb.
  • All cells executed with visible outputs (don't clear outputs).
  • Each code cell preceded by a markdown cell with your LLM prompt and fix note.
  • No API keys or secrets anywhere in the notebook.
  • Submit the GitHub link in Canvas. If it's not in the repo, it doesn't exist for grading.
Start this in Thursday lab. We'll work through the pipeline together: extracting embeddings, building the ChromaDB collection, and running the first retrieval query. Bring your dataset from Module 3 — you'll use the same images.

Overview

A three- to five-minute live demo during Tuesday's class (April 21, 7:00 PM). Your notebook is your demo — no separate slides required. Walk the class through your retrieval system, show the evaluation results, and make a live modification.

What to Cover

  1. The retrieval system: Query with a new image (one you haven't shown before). Display the query and the top 5 results. Walk the class through whether the results make visual sense.
  2. The confusion heatmap: Show Cell 6. Which two classes does the model confuse most? Pull up example images from both classes and explain the visual similarity that causes confusion.
  3. The HOG comparison: Show precision@5 for MobileNetV2 vs. HOG and the dual t-SNE plot. Explain the gap in one sentence.
