Canvas Module Setup Guide

Instructor reference. NOT for students. Step-by-step instructions for building Module 1 in Canvas.

Step 0: One-Time Course Setup

Do these once before building any modules.

A. Set Up Assignment Groups (for grade weights)

Canvas path: Assignments → top-right gear icon → Assignment Groups

Create these groups with these weights. This matches your syllabus exactly:

Group NameWeight
Participation20%
GitHub Repos25%
In-Class Presentations15%
Demos25%
Final Reflection5%
Final Portfolio Presentation10%
Important: After creating the groups, go to Assignments → gear icon → check "Weight final grade based on assignment groups." Otherwise the weights won't apply to the final grade.

B. Build Your Reusable Rubrics

Canvas path: Rubrics (left sidebar) → + Rubric

Create three rubrics you'll reuse across the course. You can attach them to assignments later.

Rubric NameCriteriaUsed For
Repo Rubric Notebook Runs (/25, standard), Task Requirements (/40, changes weekly per assignment sheet), Reflection (/20, standard), Repo Hygiene (/15, standard, no active secrets policy) Weeks 2–7 repo submissions
Presentation Rubric Content clarity (30%), Technical understanding (30%), Insight/reflection (25%), Time management (15%) Weeks 2, 3, 6 presentations
Demo Rubric Working code (30%), Live explanation (25%), Technical depth (25%), Q&A handling (20%) Weeks 4, 5, 7 demos
Tip: In Canvas, rubrics can be scored as points or left unscored for qualitative feedback. Unscored rubrics work well as structured feedback; you can then enter the final grade manually.

C. Hide Unused Navigation Items

Canvas path: Settings → Navigation tab

Drag these to the hidden section so students see a clean sidebar: BigBlueButton, Collaborations, Proctoru, LockDown Browser, Chat, Tutor.com, Course Analytics, Panorama, Simple Syllabus, Studio, Insights, Lucid, YuJa, Parchment Badges, Microsoft Education. Keep visible: Home, Announcements, Assignments, Discussions, Grades, People, Pages, Files, Simple Syllabus, Modules, Quizzes, Rubrics. Hide the built-in "Syllabus" tab (the actual syllabus lives in Simple Syllabus).

Canvas HTML tip: When pasting into the Canvas HTML editor, paste only the content inside the <body> tags, including the <style> block. Don't include the <!DOCTYPE>, <html>, <head>, or <body> tags themselves. Canvas provides its own wrapper. The inline <style> block should go at the very top of the HTML editor content.

Step 3: Import the Survey Quiz

Canvas path: Settings → Import Course Content → Content Type: "QTI .zip file" → Choose File: 01-survey-quiz.qti.zip → Import

After import, go to Quizzes and find "Student Background & Tech-Vibe Survey." Edit it:

  1. Set Quiz Type to Graded Survey (this awards completion points regardless of answers)
  2. Assign it to the Participation assignment group
  3. Set Available from / Until dates as appropriate
  4. Preview it to make sure all 40 questions look right

Step 4: Create the Discussion

Canvas path: Discussions → + Discussion
  1. Title: "Ethics Thread: Where Does the Data Come From?"
  2. Switch to the HTML editor and paste the content from 07-discussion-ethics-week1.html (body content only)
  3. Check "Graded": assign to the Participation group
  4. Points: 10 (or whatever slice of participation you want)
  5. Check "Users must post before seeing replies" (prevents students from copying each other)
  6. Set Available from: March 25 / Due: March 27 at 6:00 PM (initial post) / reply due: March 31 at 11:59 PM
Tip: Canvas only lets you set one due date on a Discussion. Set it to the initial post deadline (March 27, 6 PM). Mention the reply deadline in the prompt itself and grade replies when you grade the initial posts.

Step 7: Module Settings & Prerequisites

In the Module: Click the three-dot menu on the module header → Edit
  1. Lock until: Set to the date you want students to access it (e.g., March 24 if you want them to start the day before first class).
  2. Prerequisites: For future modules (Module 2, etc.), you can require Module 1 to be completed first. For Module 1, no prerequisites needed.
  3. Requirement: Optionally, you can require students to complete items in order (sequential). This ensures they take the survey before jumping to Week 2 assignments. Click "Add requirement" and mark the survey as "must submit."

Step 8: Publish

Nothing is visible to students until you publish. When you're ready:

  1. In the Module, click the cloud icon next to each item to publish individual items, or
  2. Click the three-dot menu on the module header → "Publish All Items and Module" to publish everything at once
Pre-flight check: Before publishing, use Student View (Settings → Student View) to see exactly what students will see. Make sure all pages render correctly, the quiz works, and the assignment submission types are right.
