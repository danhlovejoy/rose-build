Deliverable 1: Lab 1, Context Engineering in Practice Lab

Overview

Every option follows the same structure: you'll run a task through four prompting approaches (zero-shot, few-shot, context-engineered, LLM-refined) using the Gemini API, compare results, and write a reflection connecting your findings to the readings. The difference is the task itself. Pick the one that interests you most.

Choose Your Task

Option A: Fact-Check the LLM
Assemble 10-15 claims (true, false, half-true, and fabricated) and ask Gemini to verdict each one. The fabricated claims are especially revealing: zero-shot prompts accept plausible fake citations at face value, while context-engineered prompts with explicit instructions about fabrication patterns catch them. Connects directly to Week 6 (hallucination testing).

Option B: Tone Detector for Social Media
Classify social media posts by tone: sarcastic, sincere, promotional, rage-bait, or informational. Tone is harder than sentiment because it's subjective and context-dependent. A sarcastic tweet and a sincere tweet can both be "negative" in sentiment, but the model needs to distinguish what the author is doing. The ambiguous cases produce the most interesting presentations.

Option C: Structured Data Extraction
Turn messy unstructured text (job postings, event listings, or recipe descriptions) into clean JSON. Extraction has binary success metrics: the JSON either parses or it doesn't, the salary is correct or it isn't. You'll see context engineering earn its keep through explicit schemas, type rules, and handling instructions for missing data.

Each option page includes the full task description, dataset guidelines, and a complete code scaffold. Pick one and follow those instructions.

What All Options Share

Regardless of which task you pick, your notebook must include:

  1. Setup and dataset (10-15 examples with your own labels for comparison). Your labels stay in a Python variable and are only used after the API returns its answer. They're never sent to the model in the prompt.
  2. Zero-shot and few-shot baselines run on the full dataset
  3. A context-engineered prompt using the anatomy-of-a-prompt framework from the readings: system instruction, task definition, output schema, calibration examples, and constraints
  4. LLM-assisted refinement where you feed Gemini its errors and ask it to diagnose and fix the prompt
  5. Comparison table with accuracy across all four approaches
  6. Reflection (200-400 words) addressing:
    • How much did structured context improve over the baselines? On which examples?
    • What did the LLM change when it refined its own prompt? Was the change something you'd have thought of?
    • How do your results connect to the readings on context engineering?
    • What would you change if you ran this experiment again?

Submission Requirements

  • All output cells visible (don't clear outputs; I need to see your results)
  • No API keys or secrets anywhere in the notebook
