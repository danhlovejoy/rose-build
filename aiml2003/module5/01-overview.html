What's in This Module

  1. Readings & Prep: A video on LLM hallucination patterns, a research summary on bias in language models, and an introduction to evaluation frameworks. Together, these explain what goes wrong inside LLMs, how to measure it, and why evaluation design is as important as model design.
  2. Module 5 Lab: Evaluating LLMs. Build a factual accuracy test (20 questions, LLM-as-judge scoring), a profession-gender association probe, a sentiment bias test across demographic groups, and a combined bias scorecard visualization.
  3. Combined Presentation: The Human Cost of Data. For dual-enrolled students. A single presentation covering both the NLP bias evaluation and the CV demographic accuracy audit.
  4. Assignment Sheet: Deliverable details, abbreviated rubrics with Canvas links, and the due date for Tuesday, April 28.
  5. Ethics Discussion: This week's ethics prompt on who should be responsible for auditing AI systems.
Tip: This module is more about test design than code. The hardest part is writing good evaluation questions — ones that are unambiguous, verifiable, and targeted at the model's likely failure modes. Spend time on your question set before Thursday lab. The code to run the evaluation is straightforward; the questions are what make the difference.
