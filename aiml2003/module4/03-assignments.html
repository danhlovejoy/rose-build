Deliverable 1: Module 4 Lab Lab

Overview

In the readings and in class, you learned how RAG combines retrieval with generation: relevant passages are pulled from a document, injected into a prompt, and used by the LLM to produce an answer grounded in evidence. This lab puts that into practice. You'll extract text from a PDF, chunk it, embed the chunks with all-MiniLM-L6-v2, store them in ChromaDB, and wire retrieval to Gemini so the model answers questions using only the document's content.

The hallucination test in Cell 10 is where the system earns its keep. A well-built RAG pipeline should refuse to answer questions outside its context. A poorly built one will fabricate answers with the same confidence it uses for correct ones. The difference is your system prompt.

What to Submit

  • A completed Colab notebook (.ipynb) saved to your GitHub repo at week5/lab4_rag.ipynb.
  • All cells executed with visible outputs (don't clear outputs).
  • Each code cell preceded by a markdown cell with your LLM prompt and fix note.
  • No API keys or secrets anywhere in the notebook.
  • Submit the GitHub link in Canvas. If it's not in the repo, it doesn't exist for grading.
Start this in Thursday lab. We'll work through the full pipeline together: PDF extraction, chunking, ChromaDB setup, and the first Gemini call. Bring a PDF you want to use, or we'll work with the default document. Come with the readings done so you know what RAG is before we build one.

Deliverable 2: Module 4 Demo Demo

Overview

A three- to five-minute live demo during Tuesday's class (April 21, 5:30 PM). Your notebook is your demo — no separate slides required. Walk the class through your RAG pipeline, ask a live question, and show the hallucination test results.

What to Cover

  1. The pipeline (30 seconds): Show your document, chunk count, and ChromaDB collection size. Orient the audience on what the system knows.
  2. A live question: Type a new question into ask() and run it. Walk through the retrieved chunks and the generated answer. Did the model ground its answer in the context?
  3. The hallucination test: Show Cell 10 results. If the system hallucinated, explain why. If it refused correctly, show the system prompt that prevented fabrication.
  4. Live modification: Change something — chunk size, number of retrieved chunks, or the system instruction. Re-run and explain what changed. This is worth 25 points on the demo rubric.

Format

Share your screen and run your notebook live. If you attend via Zoom, have your notebook open and ready to run before your turn.
