Deliverable 1: Module 3 Lab Lab

Overview

In the readings and in class, you learned that sentence transformers encode entire sentences as dense vectors, placing semantically similar sentences close together in 384-dimensional space regardless of surface-level word overlap. This lab puts that into practice. You'll generate embeddings for a dataset of sentences using all-MiniLM-L6-v2, build a semantic search system using cosine similarity, compare it directly against your Module 2 TF-IDF baseline, and use t-SNE to visualize what the embedding space actually looks like.

The comparison with Module 2 is the point. TF-IDF can't recognize that "I loved it" and "absolutely fantastic film" belong together; the embedding model can. Finding where the improvement is real, and finding where it isn't, is what the lab is about.

About This Lab

This lab has no code scaffold. You write every cell yourself, using an LLM to help you generate the implementation. The lab sheet gives you specification boxes: each one tells you exactly what a cell must do, what it takes in, and what it must produce. Your job is to give those specifications to Gemini (or another LLM) and turn the output into working code.

For each code cell you generate, include a markdown cell documenting the prompt you used and one sentence describing any fix you had to make. This process is the core skill of the module.

What to Submit

  • A completed Colab notebook (.ipynb) saved to your GitHub repo at week4/lab3_embeddings.ipynb.
  • All cells executed with visible outputs (don't clear outputs; I need to see your results).
  • Each code cell preceded by a markdown cell with your prompt and fix note.
  • No API keys or secrets anywhere in the notebook.
  • Submit the GitHub link in Canvas. The notebook in GitHub IS your submission — if it's not in the repo, it doesn't exist for grading.
Start this in Thursday lab. We'll work through the embedding pipeline together: loading the model, generating vectors, building the similarity search. Come with the readings done so you already know what a sentence transformer is doing before you write the first cell.

Deliverable 2: Module 3 Demo Demo

Overview

A three- to five-minute live demo during Tuesday's class (April 14, 5:30 PM). Your notebook is your demo — no separate slides required. Walk the class through your results, make a live modification, and explain what changed.

What to Cover

In three to five minutes, walk the class through a live demonstration:

  1. The win: Show one query where semantic search found genuinely similar sentences that TF-IDF missed. Navigate to that cell in your notebook and explain why the embedding model got it right.
  2. The t-SNE plot: Show your visualization and point to one cluster that surprised you. Explain what the model considers "similar" that you wouldn't have expected.
  3. Live modification: Change a query or a parameter, run the cell live, and explain what changed and why. This is the core of the demo — don't skip it.
  4. Your takeaway: Based on your results, what does the shift from TF-IDF to embeddings actually buy you, and what does it cost?

Format

Share your screen and run your notebook live. If you attend via Zoom, be prepared to share your screen and have your notebook open and ready to run.

Tip: The t-SNE plot is usually the most interesting visual. Spend time with it before class and find something worth pointing to. A cluster that makes sense is less interesting than one that doesn't.
