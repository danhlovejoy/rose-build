The Task

In the readings and in class, you learned that TF-IDF represents text as sparse vectors of word counts. Every dimension is a known word; the numbers say how often it appears. That approach cannot tell that "great" and "excellent" mean the same thing, because they occupy different dimensions with no connection between them.

This week you replace word counts with dense embeddings: vectors where each dimension encodes abstract meaning learned from millions of texts. Two sentences about cooking will land close together even if they share zero words. You will build a semantic retrieval system, search it with your own queries, and then run the same texts through TF-IDF to see exactly where sparse features fail and dense representations succeed.

Your Dataset

Curated Text Collection

65 short texts across four categories: technology (15), sports (15), cooking (15), and travel (15), plus 5 cross-category edge cases. Each text is one to two sentences of factual content. The dataset is provided as a JSON file (lab3-dataset.json) on the lab resources page.

The categories were chosen so that TF-IDF and embeddings disagree on interesting examples: cooking and technology share process vocabulary ("batch," "reduce," "process"), sports and cooking share body-related terms, and the edge cases deliberately span two or three categories.

# Cell 5: Find similar texts

What it does: Defines a retrieval function that finds the most similar texts by cosine similarity, then runs it on one query text from each of the four categories. Displays results with category-match indicators.

Notes: The function uses array slicing [query_idx:query_idx+1] (not [query_idx]) to keep the query embedding 2D, which is required by cosine_similarity(). Set the query’s own score to −1 (not 0) to ensure it sorts to the bottom even if there are other very low scores. The returned top_indices and top_scores must be parallel — scores[i] is the score for text at index indices[i].
