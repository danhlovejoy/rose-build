Module 3: Learned Embeddings

AIML 2003 • Introduction to Natural Language Processing • Spring 2026

TF-IDF can't tell that "I loved it" and "absolutely fantastic film" mean the same thing. Sentence transformers can. This week you replace a word-counting baseline with a model that encodes meaning, and you find out exactly where the difference is real.

This Week's Focus

Generate sentence embeddings using all-MiniLM-L6-v2, build a semantic search system, compare it against your TF-IDF baseline on the same queries, and visualize the embedding space with t-SNE.

What's in This Module

  1. Readings & Prep: StatQuest on word embeddings, the Illustrated Word2Vec, the Sentence Transformers quickstart, and the Google Embedding Projector. Together, these explain what an embedding is, how sentence transformers build on word embedding ideas, and what the vector space looks like when you explore it interactively.
  2. Module 3 Lab: Sentence Embeddings & Semantic Search. Generate embeddings with all-MiniLM-L6-v2, build a semantic search system, compare it against TF-IDF on the same queries, and visualize the embedding space with t-SNE. No code scaffold this week: you write every cell using an LLM, document your prompts, and explain any fixes.
  3. Combined Lab: How Machines Represent Text and Images. For dual-enrolled students taking both NLP and CV. One notebook covering sentence embeddings and image embeddings side by side, submitted to both courses.

No code scaffold this week. The lab sheet gives you specification boxes: each one describes exactly what a cell must do, what it receives, and what it must produce.
