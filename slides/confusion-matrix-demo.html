A Classifier's Scorecard

You built an image classifier. It looks at photos of cats, dogs, and birds, and predicts a label for each one. Some predictions are right. Some are wrong. The confusion matrix records every outcome in one table.

Rows are the true labels — what the image actually shows. Columns are the model's predictions. Each cell counts how many images with that true label received that predicted label. Diagonal cells are correct predictions. Everything off the diagonal is an error.

Type I vs. Type II

Type I error (false positive): The model raises a false alarm. It labels an image as this class when it isn't. A spam filter that sends your real email to junk is making a Type I error.

Type II error (false negative): The model misses a real case. The image belongs to this class, but the model assigned a different label. A spam filter that lets spam through is making a Type II error.

Which error is worse depends on the application. A medical screening test that misses a disease (Type II) is far more dangerous than one that flags a healthy patient for follow-up (Type I). A self-driving car that hallucinates a pedestrian (Type I) stops unnecessarily; one that misses a real pedestrian (Type II) causes a collision. The confusion matrix shows both. The application decides which matters more.

Four Metrics from Four Numbers

The four cells from Stage 2 — TP, FP, FN, TN — are the raw ingredients. Four metrics extract different information from them. Each answers a different question about the classifier's performance on the selected class.

What each metric tells you
Accuracy uses all four cells. Precision and recall each ignore two. F1 combines the two it can't ignore. A model can have 95% accuracy and still have terrible precision or recall for a rare class. Accuracy counts total correct predictions across all classes; precision and recall zoom in on one class at a time. F1 collapses precision and recall into one number, but it punishes imbalance between them — if either is low, F1 drops.

When Accuracy Lies

A classifier that predicts "not bird" for every image gets 0% on birds. But if birds are only 5% of the dataset, the overall accuracy is still 95%. The confusion matrix exposes this immediately — an entire row of zeros where the bird predictions should be.

Scaling to 10 Classes

With three classes you can read every cell. With ten, you need a heatmap. The matrix below shows a HOG-based classifier's results on CIFAR-10 — the same pipeline you built in the lab. Dark cells mean high counts. The diagonal should be the darkest stripe; any bright off-diagonal cell means two classes are systematically confused.

The off-diagonal hot spots are the most informative cells. They tell you which classes the feature extractor cannot distinguish. In a HOG-based classifier, automobile and truck share similar edge profiles — both are boxy, horizontal shapes. Cat and dog share similar fur textures and body proportions. Those pairs light up off the diagonal.

Macro Average vs. Weighted Average

The per-class table above ends with two summary rows. Both compress ten per-class scores into one number, but they weight the classes differently.

Macro average treats every class equally: add up the ten precision scores and divide by ten. A class with 5 images counts the same as a class with 500. This makes macro average sensitive to performance on rare classes — if the model ignores a small class, the macro average drops noticeably.

Weighted average weights each class by its support — how many images actually belong to that class. A class with 500 images has 100× the influence of a class with 5. This gives a number closer to what the model does on a "typical" image. In a balanced dataset (equal class sizes), macro and weighted averages are identical.

When to use which. Report macro average when every class matters equally regardless of frequency — a medical classifier should detect rare diseases as well as common ones. Report weighted average when overall throughput matters and rare-class failures are acceptable. Most classification reports (including scikit-learn's) print both, plus support, so the reader can decide.
