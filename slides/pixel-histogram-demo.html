What is a pixel histogram?

A grayscale image is a 2D grid of numbers, each between 0 (black) and 255 (white). A histogram counts how many pixels fall at each value. A photo taken in bright sunlight piles up near 255. A photo taken in a dim room piles up near 0. Before any neural network touches an image, a histogram tells you whether the data is usable.

Why this matters for HOG. HOG features are computed from gradients — differences between neighboring pixel values. If the histogram is flat (all pixels near the same value), gradients are tiny and HOG produces garbage. Understanding the histogram is step one in diagnosing a bad feature extractor.

Think about it

Try this with your own photo: Take a picture in a very dark room and one in bright daylight. Load each one here. The histogram will tell you immediately which image has more usable information. The dark image's histogram is crushed toward 0 — there are almost no gradients for HOG to work with.

Histogram equalization sounds like magic. It is not. Equalization stretches the histogram so pixel values are more evenly distributed. It recovers detail in dark images — but it also amplifies noise. A histogram with a long flat tail of very dark pixels doesn't contain hidden detail; it contains sensor noise. Equalization will make that noise more visible, not more informative.

Discussion question for ethics: Surveillance cameras in low-light environments produce images whose histograms are heavily left-skewed. Histogram equalization can be applied automatically to "improve" them before a face detection model runs. Who decides whether that enhancement is appropriate? What is lost when the original distribution is discarded?
