One Filter Sliding Across an Image

In Module 1 you applied Sobel edge-detection kernels to photographs. A CNN does the same operation: slide a small kernel across the image and compute a weighted sum at each position. The output is a feature map where each pixel records how strongly that patch matched the kernel's pattern.

A Layer Applies Many Filters at Once

A single convolutional layer does not apply one filter. It applies many in parallel, each looking for a different pattern. The first layer of MobileNetV2 runs 32 different 3×3 kernels across the input. Each one produces its own feature map. Below, four filters run on the same image simultaneously. ReLU and Max Pooling

After convolution, two operations clean up the feature maps. ReLU (Rectified Linear Unit) zeroes out all negative values: if the kernel's pattern wasn't present, the activation is exactly zero. Max pooling then shrinks each map by keeping only the maximum value in each 2×2 block. The spatial resolution halves, but the strongest activations survive.

Why pool? Pooling makes the representation smaller (less computation in subsequent layers) and slightly translation-invariant: a feature detected at pixel (2,3) and one at (3,3) collapse into the same pooled cell. The network cares that a vertical edge exists in that region, not its exact pixel coordinate.

4

Stack the Layers

A CNN repeats the conv → ReLU → pool sequence multiple times. Each repetition shrinks the spatial dimensions and grows the channel count. Early layers detect edges. Later layers combine those edges into textures, shapes, and object parts. The table below shows what happens when you stack two convolutional layers on the 8×8 L image.

MobileNetV2 does this 53 layers deep. The input is 224×224×3 (an RGB image). After 53 convolutional layers with ReLU and various pooling operations, the output is 7×7×1280. Spatial detail has been compressed from 150,528 pixels down to 49 cells, while the channel count grew from 3 (red, green, blue) to 1,280 learned feature detectors.

5

Fully Connected: Features Become Class Scores

After the final pooling layer, the remaining feature maps are collapsed into a single vector. In MobileNetV2, global average pooling computes the mean of each 7×7 feature map, producing one number per channel: a 1,280-dimensional vector. A fully-connected layer then multiplies that vector by a weight matrix to produce 1,000 class scores (one per ImageNet category).

Our toy CNN produced an 8-d feature vector from the L-shaped input image. Below, we classify it against five possible letter shapes: L, T, I, O, and Z. Each class has its own row of 8 weights. The dot product of those weights with the feature vector determines the score. (MobileNetV2 does the same thing at scale: 1,280-d vector, 1,000 classes.) Click a class to see the computation.

Select a class above.
The FC layer is a lookup. Each of the 1,000 rows in the weight matrix encodes "what does a [class] look like in the 1,280-dimensional feature space?" The dot product measures how well the input's feature vector matches each template. The highest score wins. This is the same dot-product-as-similarity you used with cosine similarity in Module 3 and Module 4.

Strip the Classifier, Keep the Features

For image classification, you keep the FC layer and take the argmax. For everything else you have done this semester, you strip it off. The 1,280-dimensional vector before the FC layer is the embedding: a learned representation where similar images land near each other in vector space.

Python:  model = MobileNetV2(weights='imagenet', include_top=False, pooling='avg')

That one argument — include_top=False — removes the 1,000-class FC layer and returns the 1,280-dimensional feature vector. This is the line of code you ran in Module 3, Cell 1. Every retrieval query, every similarity score, every t-SNE plot you built this semester started from this vector.

The CNN is the feature extractor. Classification, retrieval, bias auditing, and captioning are different tasks built on the same foundation: a convolutional network that converts pixels into a compact vector encoding what is in the image. The architecture you stepped through in Stages 1–4 — conv, ReLU, pool, repeat — is the engine behind every lab notebook you have written this semester.
