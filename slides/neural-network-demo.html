Inside a Single Neuron

A neuron computes a weighted sum of its inputs, adds a bias, and applies a sigmoid function that squashes the result between 0 and 1. The weights and bias define a straight line. Points on one side are classified as blue; points on the other side, orange. Drag the sliders to move the line and try to separate both classes.

A Hidden Layer Untangles the Data

Adding a layer of neurons between input and output lets the network learn its own coordinate system. Each hidden neuron draws a line in the original space; the output neuron combines them to carve out a curved region. The left plot shows the decision boundary in the original space. The right plot shows the same data after the hidden layer has transformed it — each point's position is determined by two of the hidden neurons' outputs.

Watch the right plot. The hidden layer learns a new coordinate system where the two classes pull apart. In the original space the crescents interleave, but in the hidden layer's space they separate into distinct clusters. Once they're separated, the output neuron can draw a straight line between them. Each layer in a deep network does the same thing: it transforms the data so the next layer's job becomes easier.

The Representation Is the Embedding

After training, each input passes through the hidden layer and comes out as a 4-dimensional vector — one value per hidden neuron. That vector is the embedding: a learned representation that captures class-relevant structure. Strip the output neuron away and you're left with a coordinate system where similar inputs land near each other.

From 4 dimensions to 1,280. This toy network learned a 4-dimensional embedding from 200 points. MobileNetV2, which you'll use in Thursday's lab, learns a 1,280-dimensional embedding from 1.2 million ImageNet images. The principle is identical: strip the classification head, keep the representation, and use cosine similarity to find what's nearby. The only differences are scale and what the network was trained to see.
