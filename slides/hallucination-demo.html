Hallucination: Why Models Lie Fluently

AIML 2003 — Interactive Walkthrough

A language model doesn't know facts. It predicts the next token. Sometimes the most likely token is wrong.

1

One Token at a Time

A language model generates text by predicting one token at a time. At each step, it computes a probability for every token in its vocabulary and picks one. The chosen token becomes part of the context, and the process repeats. The model never "looks up" a fact — it asks "what word is most likely to come next, given everything so far?"

2

The Branch Point

Hallucination often starts at a single token. The model assigns higher probability to a wrong token than the correct one — because the wrong answer appears more frequently in training text. Once that wrong token is chosen, the rest of the sentence follows logically from the error. The continuation is fluent, specific, and confident. The only mistake was one word.

The wrong path isn't random. The model didn't produce gibberish. It produced a coherent, detailed, wrong sentence — because once "Sydney" is chosen, the most likely next tokens are facts about Sydney, and those facts are individually correct. The hallucination is in the framing, not the details.
↓
3

Temperature Changes the Odds

Before picking a token, the model divides its raw scores (logits) by a number called temperature. Low temperature makes the top token dominate; high temperature flattens the distribution so any token has a chance. Temperature doesn't make the model smarter or more accurate. It controls how adventurous the sampling is.

Adjust temperature
P(tokeni) = exp(logiti / T)  /  ∑ exp(logitj / T)
Temperature doesn't fix hallucination. At T=0.1, the model always picks the top token — but if the top token is wrong (as in Stage 1), low temperature guarantees the hallucination. At T=2.0, the correct token gets a better chance, but so does every other wrong token. Temperature redistributes probability; it doesn't add knowledge.

You cannot tell which passage is wrong by looking at the probabilities. Both passages are fluent. Both have high average token probabilities. The model generated them using the same process. Confidence is a property of the generation mechanism, not of the underlying facts. A model that has absorbed a myth from ten thousand web pages will state the myth with the same confidence it states a truth from ten thousand textbooks.
Why the wrong answer can win

Token probability comes from pattern frequency in training data. The phrase "the capital of Australia is Sydney" appears on countless web pages, travel blogs, and casual references — because people talk about Sydney more than Canberra. The phrase "humans only use 10% of their brain" has been repeated in movies, self-help books, and clickbait for decades. The model learned that these patterns are common, and common patterns get high probability. Frequency is not truth.
