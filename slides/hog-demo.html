Convert to Grayscale

Color doesn't help HOG. Edge structure is the same whether a wall is red or blue. Converting to grayscale collapses three channels (R, G, B) into one brightness value per pixel. Every step after this operates on a single 128×128 grid of numbers.

3

Compute Gradients

A gradient measures how fast brightness changes at each pixel. The horizontal gradient (Gx) catches vertical edges. The vertical gradient (Gy) catches horizontal edges. Together, they give each pixel a magnitude (how strong the edge is) and a direction (which way it points).

Gx (horizontal gradient)
Gy (vertical gradient)
Magnitude (edge strength)
Direction (edge angle, color-coded)
Reading the color-coding

Gx and Gy: Green pixels = brightness increasing in that direction. Red pixels = brightness decreasing. Dark blue = no change. Brighter colors = stronger gradients.

Magnitude: Brighter = stronger edge. Black = flat area with no brightness change.

Direction: Color = edge angle (hue cycles through 0°–180°). Saturation = edge strength. Gray pixels have weak gradients, so their direction is unreliable and shown desaturated. The color wheel: • 0° horizontal • 60° • 90° vertical • 120° • 150° • 180° wraps to 0°.

The Sobel filter is a 3×3 kernel convolved across the image. Gx uses [[-1,0,1],[-2,0,2],[-1,0,1]] to detect horizontal brightness changes. Gy uses the transpose. Magnitude = √(Gx² + Gy²). Direction = arctan(Gy / Gx).
↓
4

Divide into Cells, Build Histograms

This is the core of HOG. The image is divided into a grid of small cells — each cell is 8×8 pixels. Inside each cell, every pixel has a gradient direction (from Stage 3) and a gradient magnitude. The question is: what directions do the edges in this cell point, and how strong are they?

To answer that, HOG sorts all 64 gradient directions within each cell into 9 orientation bins, spaced 20° apart from 0° to 180°. Each pixel "votes" for a bin based on its gradient direction, and its vote is weighted by its gradient magnitude — strong edges count more than weak ones. The result is a 9-number histogram: a compact summary of the cell's edge pattern.

Click any cell in the grid below to see its histogram. The zoomed view shows the actual gradient arrows inside the cell, and the histogram shows how those arrows get sorted into bins.

How to read the histogram

Each bar represents a 20° slice of orientation space. The height of the bar is the total gradient magnitude of all pixels in the cell whose edges point in that direction. A tall bar at 80°–100° means the cell has strong near-vertical edges. A flat histogram means edges point in many directions (or the cell has no edges at all).

The zoomed cell view shows small arrows at each pixel: the arrow direction is the gradient direction, and the arrow length is the gradient magnitude. You can trace how each arrow contributes to the histogram.

The 9 orientation bins

Each bin captures a 20° wedge of the half-circle (0°–180°). The filled arc shows which gradient directions fall into that bin.

Why 0°–180° and not 0°–360°? HOG uses unsigned gradients. A dark-to-light edge and a light-to-dark edge at the same angle are the same edge — just viewed from opposite sides. Folding 360° into 180° makes the descriptor invariant to contrast polarity. A horizontal edge is a horizontal edge, regardless of which side is brighter.

5

The HOG Descriptor

Stack all 256 cell histograms into one long vector. That's the HOG descriptor — a numerical fingerprint of the image's edge structure. The classic visualization draws oriented lines in each cell, with length proportional to the bin magnitude.

HOG visualization overlay
The full descriptor vector (2,304 values)

Scroll to see all 2,304 numbers. Each group of 9 is one cell's histogram. The entire vector is what gets compared via cosine similarity.

What HOG keeps and what it throws away. The descriptor encodes local edge orientations and their magnitudes. It throws away color, exact pixel positions, and global spatial relationships. Two images with the same edge pattern but different colors produce nearly identical HOG descriptors.
